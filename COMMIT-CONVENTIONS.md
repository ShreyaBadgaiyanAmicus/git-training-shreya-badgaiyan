## Task 15: Commit Conventions Documentation

Created `COMMIT-CONVENTIONS.md` to document the commit format followed in the project. Used the Conventional Commit format:

```
type: subject
```

The commit types used include:

* `feat` – adding a new feature
* `fix` – fixing a bug or issue
* `docs` – documentation changes
* `chore` – maintenance tasks
* `refactor` – code improvements without changing functionality
* `test` – adding or updating tests

Examples of good commit messages:

* `feat: add navigation bar`
* `docs: update recovery documentation`
* `fix: resolve merge conflict issue`

Examples of bad commit messages:

* `WIP`
* `fix`
* `update files`
* `changes done`

Consistent commit messages help teams understand project history, track changes easily, review code efficiently, and identify the purpose of each change without opening every commit.
